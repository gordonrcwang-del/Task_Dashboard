## Key File Locations

**Entry Points:**

- `app.py` (line 244): Starts Flask server on 0.0.0.0:5001
  - Serves static/index.html at root
  - Routes all task CRUD operations
  - Triggers sync_tasks.py subprocess on /sync endpoint

- `static/index.html` (line 668): Frontend loads on page load
  - Calls loadTasks() → fetches /tasks → render()
  - Provides UI for task creation, completion, deletion, pool movement

- `agent_run.py` (line 142): Invoked by cron (com.gordonrcwang.agentrun.plist)
  - Fetches pending agent-pool tasks
  - Spawns Claude CLI for each
  - Marks complete and logs to agent_run.log

- `sync_tasks.py` (line 245): Invoked by /sync endpoint or external cron
  - Fetches Gmail unread (last 7 days) and Calendar events (next 14 days)
  - Creates tasks via POST /tasks
  - Cleans up stale calendar tasks

## Naming Conventions

**Files:**

- Python modules: `lowercase_with_underscores.py` (app.py, agent_run.py, sync_tasks.py)
- Frontend: `index.html` (single SPA file)
- Data: `tasks.json`, `tasks.lock`, `tasks.log`
- Credentials: `google_*.json`
- Config: `*.plist` for macOS plist

**Directories:**

- Lowercase with underscores: `deliverables/`, `planning/`, `.planning/codebase/`
- Static assets: `static/`

**Functions:**

- Python: lowercase_with_underscores
  - Public: read_tasks(), write_tasks(), run_sync(), run_task(), sync_gmail()
  - Private (prefixed with _): _spawn_next_recurrence(), _purge_old_completed(), build_prompt(), slugify()
- JavaScript: camelCase
  - Load: loadTasks()
  - Render: renderSchedule(), renderPool(), renderCard(), renderDone()
  - Action: submitTask(), moveTask(), completeTask(), deleteTask(), undoTask()
  - Sync: runSync()

**Variables:**

- Python: lowercase_with_underscores (data, creds, tasks, msg_id)
- JavaScript: camelCase (allTasks, allTasks, isToday, doneLength)

**Constants:**

- Python SCREAMING_SNAKE_CASE: RECURRENCE_DELTA, TASK_TIMEOUT, SYSTEM_CONSTRAINTS, ACTION_KEYWORDS
- JavaScript PascalCase for constants: Q_ORDER, Q_LABELS, Q_CLASS, BADGE_Q

**Task IDs:**

- Format: UUID (str(uuid.uuid4()) in Python, app.py line 144)
- Stored in task["id"]
- Used as route parameter: /tasks/<task_id>, /tasks/<task_id>/move

**Dates:**

- Format: ISO 8601 (YYYY-MM-DD for date fields, ISO 8601 full for timestamps)
- Stored as strings in JSON
- Parsed on demand in Python (datetime.fromisoformat) and JavaScript (new Date)

## Where to Add New Code

**New Feature (Task Management):**

- **Primary code:** Add endpoint in `app.py` (example: @app.route("/tasks/...", methods=["POST"]))
- **Logic:** Add helper function in `app.py` or as new module if >100 lines
- **Tests:** Create test file `test_app.py` (pattern not yet established)
- **Frontend:** Add form field to `static/index.html` form (lines 375-399) and handle in submitTask() or new handler

**New Integration (External Source):**

- **Sync logic:** Add function in `sync_tasks.py` (follow sync_gmail/sync_calendar pattern)
- **Authorization:** Store credentials in project root as `{service}_credentials.json` and `{service}_token.json`
- **Deduplication:** Use existing_source_ids() pattern (sync_tasks.py lines 67-75)
- **Task creation:** Call post_task() wrapper (lines 78-85)
- **Cleanup:** Add cleanup routine for stale tasks sourced from service

**New Agent Task Type:**

- **Agent script:** Create new Python module in project root if needed (parallel to agent_run.py pattern)
- **Task marking:** Use mark_complete(task_id) pattern (agent_run.py lines 53-60)
- **Deliverables:** Write to DELIVERABLES_DIR with naming convention YYYY-MM-DD_slug.md
- **Constraints:** Add to SYSTEM_CONSTRAINTS (agent_run.py lines 69-76) to prevent undesired actions

**Utilities:**

- **Shared helpers:** Create module in project root (no utils/ directory yet)
- **File I/O:** Use read_tasks()/write_tasks() wrappers with FileLock (pattern in app.py, agent_run.py, sync_tasks.py)
- **Logging:** Use logging.basicConfig pattern, append to tasks.log or dedicated log

**Frontend Components:**

- **New section:** Add <div id="..."> in HTML (example: #col-schedule at line 404)
- **Render function:** Add renderSection() in script block following renderPool pattern (line 519)
- **Styling:** Add CSS in <style> block following existing patterns (flexbox, color palette)
- **API calls:** Use api(path, method, body) helper (line 445)
